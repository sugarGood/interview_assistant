# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

这是一个面试准备项目，核心输入是 `简历.pdf`，核心知识库是 `题库/*.md`。项目通过自定义 skills 提供模拟面试、抽题复习、保存题目和薄弱分析工作流。

## 核心文件

- `题库/README.md` — 题库索引
- `题库/*.md` — 分方向题库文件（如 `mysql.md`、`jvm.md`、`redis.md`）
- `简历.pdf` — 用户简历，模拟面试和薄弱点分析时需要读取

## 可用 Skills

- `/mock-interview` — 模拟面试，结合简历扮演面试官提问，给出评分和反馈
- `/review-quiz` — 从 `题库/*.md` 抽题，检验记忆效果
- `/save-interview` — 将面试题保存到对应方向的题库文件
- `/weak-analysis` — 分析面试题覆盖情况，找出薄弱环节

## 核心原则（必须遵守）

- 面试方向必须由“当前简历技术栈”决定，不能被历史题库反向绑定。
- 如果简历技术栈从 Java 变更为前端 / Go / Python，出题与分析方向必须同步切换。
- 题库是持续演进的：如果出现新方向且 `题库` 下没有对应 `md`，直接创建新文件并写入。
- 历史题库只作为补充材料，不能覆盖简历优先级。

## 面试题要求

- 必须达到大厂面试标准，不能太基础
- 重原理、底层实现、排查思路，少出纯定义背诵题
- 多出"为什么"、"怎么实现的"、"出了问题怎么排查"类题目
- 保存面试题时按 `## {题目}\n{答案}` 或“讲解精炼笔记”格式写入对应方向文件（不是统一追加到单文件末尾）

## 讲解与收口约定

- 当用户说“详细讲解”时，进入自由追问式讲解，不强制固定模板。
- 用户可连续追问，直到用户明确说“下一题”。
- 用户说“下一题”时，先输出本题讲解精炼笔记，再询问是否保存到题库。
