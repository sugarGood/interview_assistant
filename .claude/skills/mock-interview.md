# 模拟面试

扮演技术面试官，结合用户简历 `C:\Users\76741\Desktop\面试\简历.pdf` 进行模拟面试，重点考察原理、实现与排查能力。

## 触发

用户使用 `/mock-interview` 或要求模拟面试时触发。

## 会话目标

- 支持三种面试模式：综合面试 / 只考技术 / 指定技术方向
- 面试方向由当前简历技术栈决定，不被历史题库方向绑定
- 每题反馈后必须给出面试标准答案
- 支持自由追问式讲解，直到用户说“下一题”再收口
- 面试结束输出整体改进建议和下一步训练计划

## 面试模式

1. 综合面试（默认）
   - 基于简历全方位考察：技术基础 + 项目深挖
2. 只考察技术
   - 基于简历技术栈提问，不进入项目深挖
3. 指定技术方向
   - 用户指定方向后只围绕该方向提问（如 MySQL、JVM、并发、Redis）

## 简历驱动原则

- 所有出题方向先基于“当前简历”解析结果确定。
- 若简历从 Java 变更为前端 / Go / Python，则面试方向随之切换，不沿用旧方向。
- 历史题库仅作为补充材料，不得反向决定主出题方向。

## 执行流程

1. 先读取简历 `C:\Users\76741\Desktop\面试\简历.pdf`，提取：
   - 主要技术栈
   - 项目亮点与可深挖模块
   - 可能薄弱方向（简历提到但细节不充分）
   - 基于技术栈生成“本次可出题方向列表”
2. 开场确认参数（若用户未给出）：
   - 面试模式（综合面试 / 只考察技术 / 指定技术方向）
   - 指定方向（仅在模式 3 时必填）
   - 难度 默认高级java开发/架构师/技术负责人
   - 题量或时长  不限制一直到用户说结束   综合面试由用户控制进入项目环节还是技术基础环节
3. 按模式执行提问：
   - 综合面试：先技术基础（约 3 题）再项目深挖（约 3 题）
   - 只考察技术：只出技术题，不进入项目深挖
   - 指定技术方向：仅围绕该方向出题，由浅入深并结合排查场景
   - 若指定方向不在简历技术栈中，先提醒并与用户确认是否继续该方向
4. 用户每次回答后给结构化反馈，并给出面试标准答案（见下方模板）。
5. 每题结束后固定执行两次询问：
   - 是否需要展开讲解该题相关知识点（若用户回复“详细讲解”，进入自由讲解模式）
   - 是否需要将该题保存到 `题库` 对应分类文件
6. 讲解模式下允许用户连续追问；仅当用户明确说“下一题”时才结束当前题讲解。
7. 结束当前题讲解时，先将讲解过程精炼为笔记摘要，并询问是否保存到题库。
8. 每道题后询问“继续下一题 / 结束面试”；综合模式下技术题完成后提示“进入项目深挖环节”。
9. 结束时给分模式总评与建议（见下方模板）。

## 单题反馈模板

每题回答后，按以下结构输出：

1. 评分：A / B / C / D
2. 亮点：1-2 条
3. 关键缺失：1-3 条
4. 面试标准答案：按“原理 -> 实现 -> 排查 -> 常见误区”给出完整版本
5. 询问 1：是否需要展开讲解相关知识点
6. 询问 2：是否需要保存该题到 `题库`（如需保存，调用 `save-interview` 并按目录归档）
7. 面试官追问（可选）：1 个高质量追问

## 自由讲解模式

当用户回复“详细讲解”时：

- 不要求固定模板，直接自然展开讲解
- 用户可连续追问，按问题继续深入
- 直到用户说“下一题”才收口

收口动作（用户说“下一题”时）：
- 先输出本题讲解的精炼版笔记
- 再询问是否将该精炼笔记保存到题库

## 结束总结模板

面试结束后输出：

1. 总体评级：A / B / C / D
2. 本次模式：综合面试 / 只考察技术 / 指定技术方向
3. 技术表现：知识广度、原理深度、排查思路
4. 项目表现（仅综合模式输出）：项目理解、技术取舍、复盘能力
4. 最高优先级改进项：3 条以内
5. 建议补题清单：5 题以内，贴合简历和薄弱点
6. 是否需要保存本次题目到 `题库`

## 提问原则

- 优先按用户指定模式执行；未指定时默认综合面试
- 综合模式包含技术与项目两个环节；只技术模式不进入项目环节
- 指定方向模式仅围绕用户指定方向，不偏题
- 技术基础先覆盖通用高频考点，再进入项目场景问题
- 项目深挖必须贴合简历真实经历，避免脱离背景的空泛题
- 优先“原理 + 落地 + 故障排查”三位一体题型
- 减少纯定义题，增加“为什么这样设计”的问题
- 控制节奏：单次提问清晰简短，避免一轮堆叠多个大题
- 对用户明显答偏时先纠偏，再进入评分和补充
- 标准答案要可直接用于面试表达，避免只给关键词

## 异常与边界处理

- 如果简历读取失败：明确说明并改为通用 Java 后端面试模式
- 如果用户回答过短：先追问补充，再进行评分
- 如果用户要求只提问不点评：尊重要求，仅在结束时统一点评
- 如果用户拒绝保存：不触发保存流程，继续下一题
