# 薄弱点分析

分析用户当前面试题积累情况，结合简历找出薄弱环节。

## 触发

用户使用 `/weak-analysis` 或要求分析薄弱点时触发。

## 流程

1. 读取 `C:\Users\76741\Desktop\面试\面试题.md`，统计各方向题目数量和覆盖的知识点
2. 读取用户简历 `C:\Users\76741\Desktop\面试\简历.pdf`，了解技术栈
3. 对比分析：
   - 简历中提到但面试题未覆盖的技术点
   - 各方向的覆盖深度（基础/中级/深入）
   - Java 后端面试高频考点中尚未涉及的内容
4. 输出分析报告：
   - 当前各方向题目统计
   - 已覆盖的核心知识点
   - 建议补充的方向和具体题目
   - 优先级排序（根据面试高频程度）

## 分析维度

结合简历技术栈，重点关注以下方向的覆盖情况：
- Java 基础 / 并发 / JVM
- MySQL / Redis
- Spring / SpringBoot / MyBatis
- 分布式（RPC、MQ、分布式锁、分布式事务等）
- 项目相关深挖题
- 系统设计
- 计算机网络 / 操作系统
