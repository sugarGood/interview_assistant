# 保存面试题

将对话中涉及的面试题内容保存到 `C:\Users\76741\Desktop\面试\题库\*.md` 的对应分类文件中。

## 触发时机

- 用户使用 `/save-interview` 命令
- 用户明确要求保存某道面试题

## 会话目标

- 以统一格式持久化题目，便于后续抽题与分析
- 尽量避免重复保存同名或高度相似题目
- 保存后给出明确回执，告知写入结果
- 题库目录结构随简历技术栈动态扩展，缺失方向直接创建新文件

## 保存格式

按以下格式写入到目标目录对应位置：

```
## {题目}
{答案内容，保留关键点和结构}
```

若用户要求保存“讲解过程精炼内容”，使用讲解笔记格式：

```
## {题目}
### 讲解精炼笔记
{本轮讲解总结，突出关键结论、易错点、排查要点}
```

## 目录归档规则

- 优先使用分目录题库：`题库/mysql.md`、`题库/java-concurrency.md`、`题库/jvm.md`、`题库/spring-springboot.md`、`题库/redis.md`、`题库/distributed.md`、`题库/system-design.md`、`题库/network-os.md`、`题库/project-deep-dive.md`、`题库/clickhouse.md`、`题库/elasticsearch.md`。
- 保存时先判断题目所属分类，再写入对应分类文件末尾。
- 若分目录题库不存在，先创建 `题库` 与对应分类文件后再写入。
- 若无法准确判断分类，先询问用户归档目录，不可默认乱放。
- 若匹配到的新技术方向在 `题库` 下没有对应 `md`，直接创建 `{方向}.md` 后写入。

- 对跨多个目录且无法唯一判定的题，默认先询问用户归档目录。

## 执行流程

1. 先检查 `C:\Users\76741\Desktop\面试\题库` 是否存在；若不存在则创建目录与默认分类文件
2. 识别要保存的题目与答案：
   - 若用户指定某一题，仅保存该题
   - 若用户要求保存本轮内容，可批量保存多题
3. 保存前做最小校验：
   - 题目不能为空
   - 答案至少包含关键点，不可只写“略”
   - 若保存讲解笔记，内容需要有明确结论，不可只有零散片段
4. 判定归档目录：
   - 按题目关键词匹配已有目录（如事务/索引归 `Mysql`，AQS/线程池归 `java并发`）
   - 对无法自动归类的题目，先询问用户选择目录
   - 若用户选择了尚不存在的新方向目录，直接创建对应 `md`
5. 做去重判断：
   - 在目标目录内优先检查同名题目
   - 若已有同名题目，默认不重复写入，先询问是否覆盖/跳过
   - 若内容高度相似，提示用户可能重复
6. 写入策略：写入对应分类文件末尾
7. 对批量保存按目录分组写入，保持各目录内部题目连续
8. 输出保存回执（见下方模板）

## 触发约定

- 当用户明确说“详细讲解后保存”“按笔记记录”“把讲解过程存一下”时，优先使用讲解笔记格式保存。
- 若用户只说“保存这题”，默认保存标准格式。

## 保存回执模板

保存后输出：

1. 保存结果：成功 / 部分成功 / 失败
2. 新增题目数：N
3. 跳过题目数：N（如重复）
4. 归档目录：`# {目录名}`
5. 目标文件：`题库/{分类文件名}`

## 异常与边界处理

- 若 `题库` 不存在：先创建 `题库` 与对应分类文件再写入
- 若用户只给了题目未给答案：先引导补全关键答案再保存
- 若用户要求“仅记录题目不记录答案”：明确标记“答案待补充”
- 若题目跨多个方向：先让用户指定主目录，只保存一份，避免多处重复
