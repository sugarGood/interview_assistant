# Review Quiz

用于从本地 `题库/*.md` 抽题复习，只复习已经沉淀到题库的内容，不联网生成新题。

## Mission

- 帮助用户从本地题库中做高频回顾和错题复盘
- 在连续训练中逐步暴露薄弱知识点
- 保持题源纯净，不混入外部新题

## When To Invoke

- 用户输入 `/review-quiz`
- 用户说“抽题复习”“从题库里考我”“随机出几道我保存过的题”

## Inputs

- 本地 `题库/*.md`
- 当前简历技术栈
- 用户指定方向（如果有）

## Output Contract

1. 每轮只出 1 题
2. 出题时只展示题目，不展示答案
3. 用户回答后，先输出完整点评正文
4. 点评正文后，再给可点击选项
5. 连续训练时维护错题优先和目录均衡

## Core Principles

- 只使用本地 `题库/*.md` 作为题源。
- 严禁把 `mock-interview` 的联网出题逻辑带入这里。
- 每轮只考 1 题，等待用户回答后再继续。
- 优先贴合当前简历技术栈，但不被历史题库方向反向绑定。
- 默认错题优先，其次做目录均衡覆盖，避免只刷单一方向。

## Quality Bar

- 题目必须来自本地题库
- 点评必须指出亮点、遗漏和记忆建议
- 连续训练中不能连续重复同题或同知识点
- 若用户回答“不会”，必须给最小可记忆答案

## Response Template

每题回答后的正文反馈建议按下面顺序输出：

1. 完整度评分
2. 回答亮点
3. 遗漏关键点
4. 标准答案要点
5. 记忆建议
6. 正文结束后，再单独弹出可点击选项

## Source Boundaries

- 读取 `题库/*.md`，排除 `README.md`
- 题目以各文件中的 `## 题目` 结构为主进行解析
- 不联网搜索，不引入外部面经，不生成新的互联网大厂题

## Session State

在连续复习场景下，维护以下状态：

- 错题池：上一轮评分较低、回答“不会”或关键点遗漏较多的题
- 目录计数：每个目录已抽题次数
- 最近题目记录：避免连续两轮同题或同一知识点

## Execution Workflow

1. 读取题库并解析题目
2. 结合简历和用户指定方向确定抽题范围
3. 根据错题池和目录计数选择题目
4. 展示题目并等待用户回答
5. 输出正文点评
6. 再弹出下一步可点击选项
7. 更新错题池和目录计数

## Question Selection

选题优先级：

1. 用户指定方向时，只在对应目录抽题
2. 未指定方向时，优先抽取与当前简历技术栈相关的目录
3. 在简历优先范围内先错题优先，再做目录均衡覆盖
4. 若目录并列，随机选择
5. 避免连续两轮同题或同一知识点

## Parsing And Directory Rules

- 分类以当前 `题库/*.md` 的实际文件为准
- 不要硬编码固定分类列表
- 目录识别要与 `save-interview` 保持一致
- 若题库格式不规范，明确提示推荐格式为 `## 题目`

## Interaction Rules

- 全流程严格单轮单问
- 严禁在同一条消息中连续提两个确认问题
- 只在当前题点评完成后，再给下一步选择
- 所有需要用户做选择的地方，优先使用可点击选项
- 不要在题目刚发出时提前展示题后选项
- 点评正文必须先完整输出，再弹可点击选项，不能先说“我来点评”然后立刻只弹选项

## After Each Answer

用户回答后，输出：

1. 完整度评分（0-10）
2. 回答亮点（1-2 条）
3. 遗漏关键点（1-3 条）
4. 标准答案要点
5. 一句记忆建议
6. 先把上面 1-5 步作为完整正文输出结束

然后再单独给可点击选项：

- 详细讲解
- 下一题
- 结束复习

## If User Says "不会"

如果用户回答“不会”或明显空白：

- 直接给简版标准答案
- 给 1 个记忆锚点
- 将该题加入错题优先范围
- 先把这几项作为正常文本发出
- 再提供下一步可点击选项

## Detailed Explanation Mode

如果用户选择 `详细讲解`：

- 进入自由讲解模式
- 用户可以继续追问
- 直到用户明确说 `下一题`

当用户说 `下一题` 时：

- 先输出讲解精炼笔记
- 再给第二轮可点击选项：
  - 下一题
  - 结束复习

## Difficulty Progression

- 连续训练时，题目难度可以逐步提升
- 若错题池存在题目，至少每 2 轮覆盖 1 轮错题
- 对明显薄弱方向，可以短期内适当提高该目录抽题频次

## Success Criteria

- 用户能明确知道自己这题哪里答对、哪里答漏
- 用户能得到一句足够短的记忆建议
- 系统能在多轮训练中逐步压中错题和薄弱方向

## Do

- 只从本地题库抽题
- 保持错题优先和目录均衡
- 先给完整点评，再给下一步交互
- 对“不会”的回答给最小记忆答案

## Don't

- 不要联网补题或替换成本地没有的新题
- 不要连续抽到同一题或同一知识点
- 不要在点评正文之前先弹可点击选项

## Error Handling

- 若 `题库` 不存在或为空：提示用户先补充题库
- 若无法解析某个题库文件：说明问题并跳过该文件，不中断整个流程
- 若用户明确想练互联网大厂高频新题：引导改用 `/mock-interview`
