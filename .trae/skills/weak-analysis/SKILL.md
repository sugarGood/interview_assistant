# Weak Analysis

用于分析当前题库覆盖情况、简历匹配度和高频缺口，找出薄弱环节并给出补题优先级。

## Mission

- 看清“简历重点”和“题库覆盖”之间的真实差距
- 判断哪些方向是短期必须补、哪些是次优先、哪些属于进阶扩展
- 输出可直接执行的补题建议，而不是泛泛而谈的总结

## When To Invoke

- 用户输入 `/weak-analysis`
- 用户说“分析我的薄弱点”“看看我哪些方向没覆盖”“给我补题建议”

## Inputs

- `简历.docx`
- 本地 `题库/*.md`
- 联网高频题信号

## Output Contract

1. 必须先给覆盖总览
2. 必须明确列出简历匹配差距
3. 必须明确列出市场高频缺口
4. 必须按 `P0 / P1 / P2` 给优先级
5. 必须给出建议新增题目和建议新建目录

## Core Principles

- 这是分析流程，不是出题流程
- 可以联网看高频信号，但用途仅限覆盖分析和补题建议
- 不能直接把分析流程替代成 `/mock-interview` 或 `/review-quiz`
- 分析方向优先由当前简历技术栈决定
- 输出重点是“差距、优先级、补题建议”，不是泛泛而谈的总结

## Quality Bar

- 不只报“题目数量”，还要判断深度和结构是否合理
- 不只说“你要多练”，而要给具体补题方向和题干建议
- 不把联网高频信息当作出题来源，而只作为覆盖分析参考
- 分析报告要能直接指导用户下一轮补题或练题

## Analysis Flow

1. 读取 `题库/*.md`，排除 `README.md`
2. 统计各方向题目数量与覆盖深度
3. 读取简历，识别技术栈、项目重点和常见考察方向
4. 联网查询高频信号，识别当前市场高频考点
5. 做差距分析：
   - 简历中提到但题库未覆盖的方向
   - 题库有数量但深度不足的方向
   - 大厂高频但当前明显缺失的方向
   - 简历出现但尚未建立题库文件的新方向
6. 输出优先级建议和补题清单

## Execution Workflow

1. 扫描题库文件和题目分布
2. 读取简历并抽取技术栈和项目重点
3. 联网收集高频题型信号
4. 做覆盖、深度、频率三维分析
5. 形成报告并给出补题优先级

## Analysis Dimensions

结合简历和题库，动态关注以下两类方向：

- 简历中明确出现、但题库覆盖不足的方向
- 题库中已经存在、但深度不足或结构失衡的方向

输出时不要把当前项目已有题库文件名写成固定规则，而要基于运行时读取结果动态生成方向列表。

## Output Structure

输出时优先包含：

- 覆盖总览：方向 -> 题目数 -> 覆盖深度
- 简历匹配差距
- 高频缺口
- 优先级清单：P0 / P1 / P2
- 建议新增题目
- 目录建议：需要新建的题库文件名

## Report Template

可按下面结构组织：

1. 覆盖总览
2. 简历重点但题库不足的点
3. 市场高频但当前缺失的点
4. P0 / P1 / P2 优先级建议
5. 每个方向建议新增 2-3 题，题干优先偏原理和排查
6. 缺失方向对应的建议题库文件名

## Report Style

- 优先写清“差距在哪里”，再写“怎么补”
- 用 `P0 / P1 / P2` 明确优先级，不要只给泛泛建议
- 建议新增题目尽量偏原理、实现和排查，不要只给定义背诵题

## Success Criteria

- 用户能一眼看出当前最该补的方向
- 用户能得到清晰的 P0 / P1 / P2 顺序
- 用户能基于报告直接决定是去 `/review-quiz` 复习，还是去 `/mock-interview` 练新题

## Do

- 先做覆盖分析，再给补题建议
- 同时考虑简历相关性和市场高频程度
- 给出能直接执行的补题方向和题干建议

## Don't

- 不要把分析流程直接变成出题流程
- 不要只报题目数量，不判断深度
- 不要只说“你需要加强某方向”而不给具体建议

## Priority Rules

- 高频程度越高，优先级越高
- 与简历项目越相关，优先级越高
- 现有覆盖越薄弱，优先级越高
- 若同时属于国内大厂高频题型，优先级继续上调
- 纯定义背诵题占比过高时，优先补原理和排查型题目

## Interaction Rules

- 如需用户补充目标岗位或优先方向，严格单轮单问
- 需要用户做选择时，优先使用可点击选项
- 分析完成后，如用户要练题：
  - 复习已有题目，引导到 `/review-quiz`
  - 练高频新题，引导到 `/mock-interview`

## Error Handling

- 若 `题库` 为空：说明当前无可分析题库，并给初始化建议
- 若简历无法读取：改为通用岗位基线分析，并明确说明
- 若某方向题目很多但深度不足：标记为“数量覆盖、深度不足”
- 若用户中途改成“直接出题”：先完成当前分析，或建议切换到对应流程
