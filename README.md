# 面试助手快速上手

这是一个用于面试准备和模拟面试的本地助手项目。

核心输入：

- `简历.docx`：用于综合面试、项目深挖、薄弱点分析
- `题库/*.md`：用于抽题复习、保存题目、分析覆盖情况

## 你可以用它做什么

1. 模拟面试
   - 先确认面试模式、方向、难度、时长和点评方式
   - 联网搜索近年互联网大厂高频题型后出题；综合面试结合简历项目，技术面试默认只按技术方向出题
   - 每次只问 1 题
   - 你回答后会得到评价、标准答案，并看到首轮固定选项：`下一题 / 详细讲解 / 保存题库（然后进入下一题）`
   - 如果你先选了`详细讲解`，讲解结束后再选：`下一题 / 保存题库（带上详细讲解，然后进入下一题）`

2. 抽题复习
   - 只从已有本地题库里抽题
   - 适合日常记忆复盘和错题回顾

3. 保存题目
   - 把当前面试题保存到对应方向题库
   - 保存前会检查是否有相似题，避免无脑重复追加

4. 薄弱点分析
   - 对比你的简历、当前题库和市场高频信号
   - 找出高频缺口、薄弱方向和补题优先级
   - 只做分析和建议，不直接进入出题

5. 使用帮助
   - 适合第一次使用时快速了解项目怎么用
   - 可以直接查看常用功能和示例命令

## 快速开始

1. 把你的简历放在项目根目录，文件名保持为 `简历.docx`
2. 确认 `题库` 目录中已经有你当前需要的题库文件
3. 如果要用 `/mock-interview`，确保当前环境可以联网
4. 选择一种使用方式开始训练
5. 如果是第一次用，优先从模拟面试或抽题复习开始

## 常用方式

### 0. 查看帮助

如果你是第一次使用，建议先看帮助。

可直接说：

- `/help`
- `这个面试助手怎么用`
- `告诉我使用方法`

### 1. 综合面试

适合完整模拟真实面试，包含技术考察和项目深挖。

可直接说：

- `/mock-interview`
- `开始综合面试`

特点：

- 会先让你确认模式、方向、难度、时长/题量、是否即时点评
- 会联网搜索互联网大厂高频题型，再结合 `简历.docx` 出题
- 技术部分更像技术负责人/架构师面
- 项目部分更像二面项目深挖
- 不参考本地 `题库/*.md` 作为出题题源

### 2. 技术面试

适合只练某一个方向。

可直接说：

- `/mock-interview 技术面试 Redis`
- `/mock-interview 技术面试 MySQL 深入追问`
- `来一场 JVM 技术面，标准一面`

技术面试可选难度/深度：

1. `基础加强`
2. `标准一面`
3. `深入追问`
4. `面试官压力面`

技术面试同样会先确认参数，再联网搜索该方向的大厂高频题型后出题，默认不参考项目经历。

### 3. 抽题复习

只适合复习你已经沉淀在本地题库里的内容，不会联网找新题。

可直接说：

- `/review-quiz`
- `从题库里抽题考我`
- `抽 Redis 方向的题`

### 4. 保存当前题目

可直接说：

- `/save-interview`
- `保存这题`
- `把这道题存到题库`

### 5. 分析薄弱点

这个模式会分析“简历要求”和“现有题库覆盖”的差距，必要时参考外部高频信号，但不会直接开始出题。

可直接说：

- `/weak-analysis`
- `帮我分析一下薄弱点`
- `看看我现在题库缺什么`

## 使用过程会发生什么

以模拟面试为例：

1. 助手先确认面试模式
2. 然后依次确认方向、难度、题量/时长、是否即时点评
3. 如果需要你做选择，会把选项明确列出来
4. 参数确认完后，会联网搜索该方向的互联网大厂高频题型
5. 每次只问你 1 个问题
6. 你回答后，会给你评价和标准答案
7. 然后会直接给你首轮固定选项：`下一题 / 详细讲解 / 保存题库（然后进入下一题）`
8. 如果你先选`详细讲解`，讲解结束后再进入二轮选择：`下一题 / 保存题库（带上详细讲解，然后进入下一题）`

## 适合新手的推荐用法

如果你第一次使用，建议从下面任意一种开始：

1. `/mock-interview`
2. `/mock-interview 技术面试 Redis 标准一面`
3. `/review-quiz`
4. `/weak-analysis`

## 一句话说明

你不一定非要记住命令，也可以直接用自然语言说你的需求，例如：

- `开始综合面试`
- `来一场 MySQL 技术面，深入追问`
- `抽几道并发题考我`
- `分析一下我现在的薄弱点`

## 三种核心模式怎么选

- 想练互联网大厂高频新题：用 `/mock-interview`
- 想复习你自己题库里已经积累的题：用 `/review-quiz`
- 想看自己还缺哪些方向和题目：用 `/weak-analysis`
